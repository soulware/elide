## Major changes

### LIST elimination — complete

The [list-elimination plan](../plans/list-elimination-plan.md) executed end to
end (#392–#406).  The coordinator no longer holds a `ListBucket`
grant at all.

- **P1** — event-log spine: windowed HEAD (last N signed records,
  0-hop common case), HEAD-CAS-before-record emit.
- **P2** — snapshots as an event projection: single User
  `snapshots/LATEST` pointer; handoff reuse driven off the CAS'd
  `names/<name>` record.
- **P3/P4** — per-volume segment HEAD (`by_id/<vol>/HEAD`):
  whole-object read-modify-write per active drain tick; reaper folded
  into the tick loop, making it the sole writer; rebuild invariant
  proptested (rebuilt HEAD is a safe superset of live state).
- **P5** — `ListBucket` grant and all runtime callers dropped (#406).

### Domain-typed store layer

`docs/design/domain-store.md` plus implementation (#407–#420): the
coordinator's raw S3 surface is wrapped in domain-typed stores —
event journal, name claims, volume data segments, HEAD metadata —
with typed verification proofs (`Verified` auth receipts, verified
leaves) replacing ad-hoc bool/string plumbing.

### Per-volume credential routing

The long tail of moving every S3 access onto the right mint role:

- `by_id` reads on `volume-ro`, writes on `volume-rw` (renamed from
  `coord-data`); cross-coord pubkey and meta reads on `coord-ro`
  (renamed from `coord-base`); shared metadata moved under a `meta/`
  prefix.
- Misroutes fixed across release, force-release, prefetch, claim
  recovery, peer discovery and volume-events listing — each a silent
  Tigris 403 until found.
- Perf recovered: cached `volume-ro` mints, warmed `coord-ro`,
  drain/GC skipped on readonly volumes.

### Mint hardening, first pass

- Mint state moved into a Tigris bucket (#451); dual-listen
  transport (#453); approval MAC + keyring (#454); template-seal
  implementation (#456); audit tags on corrupt approvals (#459);
  bootstrap enrollment renamed to invite (#449).

### Coordinator property-test expansion

Multi-volume simulation (#423), write-zeroes multi-block (#443),
local reclamation (#445), read-after-drain (#447), role-routing pins
for untested ops (#448); the CI proptest lane narrowed to what it
actually exercises (#460).

### Central auth service — design opened

`docs/design/auth-service.md` (#461): a central auth service issues
operator sessions and per-operation discharges.  The local
operator-token PoC is removed in favour of it (#462).
