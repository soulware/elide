## Major changes

### Operator authorisation wired through mint

The auth-service design landed last week is now implemented on the
mint side and proven end to end with the mint CLI as first client
(#465–#483):

- **TPC machinery** — caveat sum type (#470), keyring/MAC/approval
  storage (#471), third-party-caveat issuance (#472) and verification
  (#474), admin macaroon (#475).
- **Verify/clear split on the wire** — `verify_and_clear` returns a
  cleared bundle (#477); discharge acquisition gates the write role
  (#478).
- **Operator surface** — auth on a separate socket (#479), CLI
  discharge loop (#481), demo onboarding (#482) and the operator
  session flow (#483, which also settled the `X-Mint-Pop` header
  name).

### Template seal revision

Templates are now served from a sealed, cached surface rather than
live config (#484–#491): seal cache + `TemplateSet` (#489), serving
cutover (#490), and sealing as an authenticated
`POST /v1/admin/seal` daemon call with the pending-seal mechanism
retired (#491).  TPC configuration moved through core, staging and
transport in the same arc (#485–#487).

### Docs

- Auth-service spec refined: cached vouches, coordinator clearing,
  TPCs on role credentials, discharge scope/tier vocabulary
  (#464–#467, #480, #492).
- **`docs/design/coord-http-ipc.md`** (#469) — proposal to pivot the
  CLI ↔ coordinator socket from NDJSON+UDS to HTTP+UDS, matching
  mint; remote-coordinator IPC documented (#466).
