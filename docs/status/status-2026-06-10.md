# Elide Status — 2026-06-10

Progress since [2026-06-02](status-2026-06-02.md).  One week,
~60 PRs (#495–#555).

---

## Major changes

### Per-volume read credentials

The `volume-ro` ancestor list is retired (#521–#529): each
credential scopes to exactly one `by_id/<vol>/*` prefix and readers
obtain per-ancestor credentials over the Credentials IPC.  Target
authorization via an `AuthorizedTarget` proof type; the caveat
helper dropped in favour of `req.volume` substitution; coordinator
`volume_id` strings converged on `Ulid` (#524).

### Injection-proof template rendering

Handlebars dropped (#530): policies render by parsing the template
as JSON and substituting scalar tokens into string leaves —
injection-proof and string-positioned by construction.  Each role's
`req`/`caveat` contract is declared, sealed and enforced (#531).

### Caveat vocabulary + per-context clearing

`exp` and `NotAfter` collapsed into a single deadline caveat (#534).
Clearing is now per-macaroon rather than a flattened union:
`verify_and_clear` evaluates each macaroon in its own context and
returns a `ClearedBundle` keyed by source; discharge vocabulary
settled on `sub`/`aud` (#535/#536).

### Volume-ownership attestation

Mint tokens can now attest "this coordinator owns this volume"
(#533, #537–#542, #549, #555):

- Mint emits an attested third-party caveat for attestation roles and
  renders volume scoping from the attested discharge; `req.*` retired
  from policy templates.
- Coordinator discharge authority: rw-self and ro-ancestor predicates
  over a shared `meta/` lineage walk; coord A fetches its rw-self
  discharge before assume-role.
- The discharge side extracted into a new `elide-attestation` crate,
  with per-endpoint scheme-discriminated listeners — `[peer_fetch]`
  and `[attestation]` each take a `listen =` value and are
  presence-enabled (config break: `port`/`bind` → `listen`).
- **Design contract settled** (#555) — coord A ro-ancestor discharge
  acquisition: claim-first ordering, snapshot basis on the name
  record, `claim --force` re-owning the tail by re-signing the
  index, start anchored on the key shadow.

### Mint state, config and revocation

`[tenant]` split into `[store]` (transport-only) and `[env]`
(template-visible, sealed) (#505); credential revocation (#509–#513);
enrollment gates, shared login and a CLI-ergonomics sweep
(#495–#504, #506–#508, #516–#518); the policy-template `system.*`
namespace renamed to `mint.*` (#520).

### Mint property-test suite

~46 properties across caveat resolution, role-TTL clamping, template
injection-proofness, TPC crypto, the PoP gate, the chained-MAC
macaroon and the keyring rotation state machine (#543–#548, #551),
plus a deterministic root-key rotation-ordering integration test
(#552).

## Bug fixes

- **Partial-fork recovery** (#550) — a provisional parent recorded on
  `early_rebind` closes the #428 crash window.
- **Claim exit code** (#553) — claim exits non-zero when the
  orchestrator fails before finalize.
- **Credential resilience** — reissue-and-retry on S3 access denial
  (#532).
- `volume fetch` and its worker removed (#554) — lazy hydration on
  `volume start` covers the use case.
