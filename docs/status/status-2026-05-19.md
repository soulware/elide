# Elide Status — 2026-05-19

Progress since [2026-05-10](status-2026-05-10.md).

---

## Major changes

### mint — standalone credential-vending service

The dominant arc this window (≈30 PRs, #363–#390).  A new
macaroon-authenticated service that brokers Tigris credentials,
replacing the coordinator's in-process IAM path.

- **Standalone workspace + CI lane.**  `mint` lives in its own nested
  cargo workspace with a dedicated CI lane; clap-derived CLI;
  `./mint_data` default state dir; example configs/roles named by
  role.
- **Issuance + verification model.**  mint is issuer *and* verifier;
  capability macaroon with a PoP-signed request body
  (`elide:CoordKey` holder-of-key, RFC 7800 / RFC 9449 prior art);
  scalar-only caveats; macaroon root key is a generated `data_dir`
  file (hex), not config.
- **Bootstrap-macaroon enrollment.**  `POST /v1/enroll` with operator
  approval and interactive fingerprint confirmation; coordinator-side
  enrollment client + `[mint]` startup gate; `elide coord enroll`
  wired through the CLI.
- **Credential-per-role.**  Role authorization at the enrollment
  exchange; per-role credentials under `credentials/<role>`;
  coord-writer split per-purpose + per-volume; `coord-base`
  read-only baseline; `request.role` required, omnibus credential
  dropped.
- **Transport.**  Deployment-mode-selected listener; UDS default in
  both example configs.  600s ticket TTL with recorded rationale.
- **Reference client + tooling.**  Networked `mint client …`
  (assume-role, credential/role inspection); forward-ported real
  Tigris IAM minter behind `serve --tigris`.
- **Clean break.**  Coordinator `[iam]` / in-process Tigris path
  removed entirely (#387); `base_ro()` migrated to a read-only
  `coord-base`.

### Per-volume Tigris IAM credentials

Landed as an initial pass (#318–#324, #346–#347) before mint
superseded the in-process path: per-volume RO policy driven by the
fork ancestor list, per-coordinator writer key scoped under `by_id/`,
`Credentialer` trait abstraction.  Admin keys documented as
org-global root.  Now subsumed by mint.

### Coordinator classifier refactor

The coordinator's ad-hoc state inference replaced with typed
classifiers (#334–#340):

- `VolumeLifecycle::resolve` as the verb-facing local classifier;
  claim/fetch/stop/remove/release/start all migrated.
- `OwnershipPosition` — bucket-side classifier; claim, fetch,
  release, `hydrate_or_route`, `status_remote` migrated.
- `Role` — verb-dispatch projection of `OwnershipPosition`; verb
  gates dispatch on `Role` instead of raw position.  Dead
  `volume.draining` machinery removed.

### stop-snapshot lifecycle

`auto-snapshot` renamed to `stop-snapshot` and the
stop/release/resurrection path tightened:

- Auto-snapshot published on volume stop; release `--force` promotes
  a dead owner's auto-snapshot via server-side copy; stop skips its
  auto when already covered.
- Volume signing keys shadowed for writable resurrection;
  readonly→writable transitions supervised; auto-snapshot cleanup
  deferred until the volume signals ready.
- Remotely-owned volumes tracked via `data_dir/remote/` breadcrumbs
  and hydrated on `volume start`; supervisor liveness (not just
  inode) used to detect dir recreation.

### Peer-fetch hardening

- Lineage verified against **local** provenance, not S3; gap-free
  fence.
- Step-4 lineage walk anchored at `url_vol_id` (a rebound child fork
  was being missed) — see `[[project_body_token_lineage_anchor]]`.
- `.body` route authenticated with the coordinator `PeerFetchToken`
  (body-token claimer gate); `coord-base` read-only baseline
  replaces the dedicated peer-fetch role.

### Operator tokens

- Stored per-coordinator in `~/.elide/tokens.toml`; `elide token
  list` / `token remove <nonce>` added.
- Real chained MAC + per-token nonce + audit logging; gate
  `Request::Remove`; reframed as S3-write gating, not per-verb.
- Operator-token + isolation design extracted to
  `design-auth-model.md`.

## Bug fixes

- **GC local-apply overwrite-loss** — fixed (#377); the proptest GC
  model now routes liveness through `segment_classify::classify_entry`
  rather than a hand-rolled hash-only check
  (`[[project_proptest_gc_model_fidelity]]`).
- **Synthesised handoff materialised locally on `NeverRan` release**
  (#359) — was lost on the local side.
- **Claim → release without a start/stop detour** (#358).
- **Volume remove widened to all non-active shapes** (#341); breadcrumb-only
  release synthesises an empty handoff when no snapshot exists.
- **Claim of a fetched local fork** restores writability or refuses
  with an actionable hint; fetch/claim idempotent on owned-by-us.
- **`stop --force`** falls back to direct CLI mode when the
  coordinator is down.
- **ublk sweep scoped to `data_dir`-stamped devices** (#320) —
  multi-coord namespace safety.
- Materialized proptest reproducers split into a `volume-invariants`
  lane (#379); axum aligned on 0.8 across `elide-peer-fetch`.

## Remaining work

The mint arc is a working prototype against a settled design, plus a
clean break from the in-process IAM path.  Production hardening of
mint (deployment, key rotation, the networked transport) and
executing the LIST-elimination plan are the next concrete pieces.

Carried-over priorities, unchanged:

1. **Boot hints + S3 repacking** — still telemetry-gated.
2. **Snapshot indexes** — acceleration structure for cold
   `Volume::open()`; partly addressed by the windowed-HEAD direction.
3. **Reclamation thresholds** — unified bin-pack still uses
   placeholder thresholds; needs empirical work on an aged volume.
4. **Portable live volume** — Phase 0 foundations in; force-release +
   recovery replay remain.
5. **ublk zero-copy** — still deferred.
