# Elide Status — 2026-06-24

Progress since [2026-06-17](status-2026-06-17.md).  One week,
~12 PRs (#596–#608).

The week deployment began: mint runs as an external dependency,
the coordinator adopts mint's credential lifecycle, and the whole
stack — mint plus a coordinator, attested per-volume credentials,
shared-key demo auth — was proven on real Fly.io.

---

## Major changes

### Coordinator on mint's credential lifecycle

With mint now a separate repo, the attested-loop e2e is adapted
to the standalone, reworked mint and its bins isolated (#596).
The coordinator adopts mint's exchange-finalize credential
lifecycle (#597), and per-volume credentials are wired via
durable enrollment intermediates (`_parent` → `_intermediate`,
#598).

### mint-for-elide on Fly.io

The mint role inventory is promoted to `deploy/mint/` (tracking
mint #21's config break), synced to the always-attest model, and
the mint design docs reconciled with always-attest (#600–#602).
Role templates carry `{{build.bucket}}`, rendered by `mint render`
(#603).

A Fly.io deployment for mint-for-elide lands as a private app
(#604): `[store].bucket` derives from `DATA_BUCKET` at build,
`MINT_CONFIG` is baked into the image so in-container commands
need no `--config`, and role templates render in the build stage.

### Shared-key demo auth — proven on real Fly

Demo auth runs without a separate auth service: mint and the
coordinator share `K_M-A`, and the coordinator self-issues operator
enrollment discharges (#606/#607).  An opt-in socat bridge reaches
the demo auth over 6PN (#605).  Discharge acquisition moves behind
a `DischargeSource` trait; the coordinator gains a Fly image,
waits for enrollment at startup instead of failing closed, reads
its config path from `ELIDE_COORD_CONFIG`, fails loudly on a
missing config, and flocks the pidfile for the single-instance
guard (#607).

This was driven end-to-end against real Fly (coord ↔
mint-for-elide): self-issued enroll → approve → roles →
`coordinator.pub` on Tigris → serving.

## Remaining work

The proposal to build the deploy binaries in CI rather than
`cargo build` inside the Dockerfile is written up (#608); the
Dockerfile still compiles mint and elide from source.  Removing
that — the silent stale-binary cache has already bitten
enrollment — is the next deploy step.

Carried-over priorities, unchanged: boot hints + S3 repacking,
snapshot indexes, reclamation thresholds, ublk zero-copy.
